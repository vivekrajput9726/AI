12. TESTING

Testing is an important phase of software development that ensures the "Synora Health AI Platform" functions accurately, securely, and efficiently under different operating conditions. The system was tested using multiple testing methods to verify functionality, integration, reliability, and user satisfaction.

12.1 Unit Testing

Unit testing was performed to verify the functionality of individual modules independently. Each module was tested separately to ensure correct processing, validation, and output generation. The testing process confirmed that all modules were functioning accurately under different input conditions.

The following authentication modules were tested individually:

  1. Login Module
  2. Registration Module
  3. OTP Verification Module

1. Login Module Testing

The login module was tested using both valid and invalid login credentials to verify authentication functionality and validation handling. Some test scenarios performed are:

  • Login with incorrect email or wrong password
  • Login with a deactivated account
  • Validation of authentication error messages
  • Successful dashboard redirection after login
  • Quick Demo Login buttons for Patient, Doctor, and Admin roles

Expected Result:

  • Valid users should be successfully redirected to their role-based dashboard
  • Invalid users should receive an appropriate error message
  • Deactivated accounts should be blocked from accessing the system

Result:

The system successfully authenticated valid users and redirected them to the correct dashboard page based on their role (Patient, Doctor, or Admin). Invalid login attempts were correctly detected, and the system displayed an "Invalid credentials" error message. Deactivated accounts were blocked with an appropriate account status message.

12.2 Integration Testing

Integration testing was performed to verify communication and data flow between connected modules of the system. The integrated modules were tested together to ensure smooth interaction, correct data transfer, successful backend processing, and proper storage of generated healthcare information.

1. AI Symptom Checker and Database Integration Testing

The AI Symptom Checker module was tested together with the database management module to verify that the generated AI analysis results were processed correctly and stored successfully in the patient's health history after analysis.

During testing, different symptom descriptions such as chest pain, fever, joint pain, and breathing difficulty were entered into the system. The Groq Llama 3.1-70B AI model analyzed these symptoms and generated condition predictions, urgency levels, and specialist recommendations. After analysis, the system automatically stored the generated results in the patient profile connected to the MongoDB database.

Expected Result:

The generated AI symptom analysis should be successfully saved in the MongoDB database and displayed correctly in the patient's health history records.

Result:

The AI model successfully generated medically relevant symptom analysis based on patient input. The generated results were transferred correctly between the AI analysis module and the database system and stored successfully in the health history without data loss or processing errors.

2. Appointment Booking and Payment Integration Testing

The appointment booking module was tested together with the Razorpay payment module and the database module to verify that appointments were correctly created, payments were securely verified, and all records were stored successfully in the system after confirmation.

During testing, a patient selected a doctor, chose an available time slot, and completed the Razorpay payment. The backend performed HMAC-SHA256 cryptographic signature verification of the payment before confirming the appointment. After successful verification, the appointment record and payment record were both saved in MongoDB, and the doctor received a real-time WebSocket notification.

Expected Result:

The appointment should be confirmed only after successful payment verification. Both appointment and payment records should be saved in the database and the doctor should be notified in real time.

Result:

The system successfully processed the payment, verified the HMAC signature, saved both records in the database, and delivered the real-time notification to the doctor within one second of payment confirmation.

12.4 User Acceptance Testing (UAT)

User Acceptance Testing (UAT) was performed to verify whether the system satisfies user requirements and provides a simple, user-friendly, and effective healthcare management experience. The system was tested under practical conditions to evaluate usability, navigation, functionality, and overall user satisfaction.

During testing, users interacted with different modules such as:

  • Patient Registration and OTP Verification
  • Doctor Search and Appointment Booking
  • AI Symptom Checker
  • Razorpay Payment Processing
  • Health Record Upload and AI Analysis
  • Video Consultation and Real-Time Chat
  • Doctor Dashboard and Appointment Approval
  • Admin Doctor Verification and User Management

Users found the registration process easy to complete and the OTP verification quick and straightforward. The AI Symptom Checker provided detailed and helpful medical suggestions that users found informative and easy to understand. The appointment booking module helped users find and book doctors efficiently with a smooth payment experience through Razorpay. The health record upload feature with AI analysis was found highly useful for understanding medical reports.

The video consultation feature successfully connected patients and doctors in real time using WebRTC. The real-time chat module was found responsive and reliable. Doctors found the appointment approval workflow and AI-assisted prescription feature easy to use. Administrators were satisfied with the real-time statistics dashboard and the doctor verification workflow.

Result:

The system successfully satisfied the expected user requirements and provided a smooth, reliable, and user-friendly healthcare management experience across all three user roles — Patient, Doctor, and Administrator.
