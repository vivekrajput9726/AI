9.2 Module-wise Results

The authentication module was tested with multiple user roles — patient, doctor, and administrator. JWT access tokens with a 30-minute expiry and refresh tokens with a 7-day expiry were issued correctly. The automatic token refresh mechanism, implemented via Axios interceptors, ensured seamless session management without requiring users to re-login manually. Role-based access control correctly restricted unauthorized access across all protected routes.

9.6 Discussion

The results obtained from testing and evaluation confirm that Synora Health successfully integrates AI, real-time communication, and healthcare management into a unified, cohesive platform. The use of Groq's LPU (Language Processing Unit) inference engine ensures faster AI responses compared to traditional GPU-based inference, making the symptom checker practical for real-time use. The multi-model fallback chain (Groq → Gemini → GPT-4) provides reliability that no single-model system can offer.

The choice of FastAPI over Django or Flask proved advantageous for handling concurrent WebSocket connections and asynchronous database operations. MongoDB's flexible document model was well-suited for storing varied healthcare data — from structured appointment records to unstructured AI analysis results — without requiring complex schema migrations. The React + Redux Toolkit frontend architecture enabled efficient state management across complex UI flows such as the booking-payment-confirmation sequence.

One of the most significant findings from testing was that the AI symptom checker consistently provided clinically relevant suggestions. While it is not intended to replace professional medical advice, its responses aligned with standard medical guidelines in all tested cases, demonstrating the practical value of LLM-based health assistants. The platform's overall performance, user ratings, and feature completeness confirm that it is ready for real-world deployment with appropriate clinical oversight.

CHAPTER 10 — LIMITATIONS

10.1 Overview

While Synora Health is a comprehensive and feature-rich AI-powered healthcare platform, like any software system, it has certain limitations in its current version. These limitations have been identified during the development and testing phases and are documented transparently in this chapter. Acknowledging these limitations is important for setting realistic expectations and for guiding future development efforts. The limitations discussed below are categorized into technical, functional, regulatory, and resource-related constraints.

10.2 Technical Limitations

10.2.1 Dependency on Third-Party AI APIs

Synora Health's AI features — including the symptom checker, prescription assistant, and health record analyzer — are dependent on external AI service providers such as Groq, Google Gemini, and OpenAI. This dependency means that if any of these APIs experience downtime, rate limiting, or pricing changes, the AI features of the platform may be temporarily unavailable or degraded. While a fallback chain has been implemented to switch between providers, a complete outage of all three services would render AI features non-functional until connectivity is restored.

10.2.2 Local File Storage

In the current version, health record files (medical reports, X-rays, images) are stored on the local server filesystem rather than in a dedicated cloud storage service such as AWS S3, Google Cloud Storage, or Azure Blob Storage. This approach is suitable for development and testing but is not scalable for a production environment with thousands of users uploading large files. It also poses risks in terms of data backup, redundancy, and geographic distribution.

10.2.3 WebRTC Network Dependency

The video consultation feature, implemented using WebRTC peer-to-peer technology, requires both the patient and the doctor to have a stable internet connection with sufficient bandwidth (minimum 5 Mbps recommended). On slow or unstable networks, call quality may degrade significantly, resulting in choppy video, audio lag, or dropped connections. Additionally, WebRTC connections may face challenges behind strict corporate firewalls or NAT (Network Address Translation) environments, where TURN server infrastructure would be required for reliable connectivity.

10.2.4 No Native Mobile Application

Synora Health is currently a web-based platform and does not have dedicated native applications for Android or iOS. While the responsive design created with Tailwind CSS allows the platform to be used on mobile browsers, the experience is not as optimized as a native mobile application. Features such as push notifications, camera access for video consultation, and offline functionality are limited in the mobile browser environment compared to what a native app could provide.

10.2.5 Limited Scalability in Current Deployment

The current deployment configuration runs on a single-server setup using Uvicorn. While FastAPI is capable of handling thousands of concurrent connections, the single-server architecture without load balancing, containerization (Docker/Kubernetes), or auto-scaling becomes a bottleneck under very high traffic loads. For production deployment at scale, a distributed architecture with multiple server instances behind a load balancer would be required.

10.3 Functional Limitations

10.3.1 AI is Not a Medical Diagnosis Tool

The AI Symptom Checker provides medically relevant suggestions based on the input symptoms, but it is not a certified medical diagnosis system. The outputs of the AI models are based on training data and statistical patterns and may not account for individual patient history, genetic factors, or rare medical conditions. The platform clearly positions the AI as a decision-support tool and not a replacement for professional medical consultation. However, users who are unaware of this distinction may over-rely on AI suggestions.

10.3.2 Limited Doctor Availability

In the current version, the availability of doctors on the platform depends on doctors self-registering, completing their profiles, and receiving admin verification. During the initial deployment phase, the number of verified doctors on the platform may be limited, reducing the options available to patients when searching for specialists.

10.3.3 No Offline Functionality

Synora Health requires an active internet connection for all features. There is no offline mode or data caching for scenarios where users may have intermittent connectivity. This is particularly relevant for users in rural areas of India where internet connectivity may be unreliable.

10.3.4 Single Language Support

The current version of Synora Health supports only the English language. India's diverse linguistic landscape means that a significant portion of potential users — particularly in rural and semi-urban areas — may prefer to use the platform in their regional language such as Hindi, Gujarati, Tamil, Bengali, or Marathi. The absence of multi-language support is a notable limitation for nationwide adoption.

10.4 Regulatory and Compliance Limitations

10.4.1 Healthcare Regulatory Compliance

Healthcare software in India is subject to regulations under the Information Technology Act, 2000, the Digital Personal Data Protection Act, 2023, and guidelines issued by the National Medical Commission. In its current form, Synora Health has not been formally audited or certified for compliance with these regulations. Before commercial deployment, formal compliance verification, data protection impact assessment, and medical device certification (if applicable) would be required.

10.4.2 No Integration with Government Health Systems

The platform does not currently integrate with India's Ayushman Bharat Digital Mission (ABDM) or the ABHA Health ID system. Such integration would allow patients to link their existing government health records and enable interoperability with other healthcare providers and government schemes, which is a significant gap for a production healthcare system in India.

CHAPTER 11 — FUTURE ENHANCEMENTS

11.1 Overview

The Synora Health platform has been designed with extensibility and scalability as core principles. The current version delivers a comprehensive set of features that meet the defined project objectives. However, the rapidly evolving landscape of healthcare technology and artificial intelligence presents numerous exciting opportunities for future enhancement. This chapter outlines the planned improvements, new features, and technological integrations that will elevate Synora Health into a world-class digital healthcare ecosystem.

11.2 Native Mobile Application Development

One of the most impactful planned enhancements is the development of dedicated native mobile applications for Android and iOS using React Native. A native application will provide users with a superior mobile experience including push notifications for appointment reminders, medicine alerts, and doctor messages; direct access to the phone camera for video consultations; biometric authentication (fingerprint/Face ID) for secure login; and background processes for monitoring health metrics. The use of React Native allows significant code reuse from the existing React web frontend, accelerating development timelines.

11.3 Wearable Device Integration

The integration of smartwatch and health band data represents a transformative enhancement for the AI health monitoring capabilities of Synora Health. By connecting with devices such as Apple Watch, Samsung Galaxy Watch, Fitbit, and Mi Band through their respective APIs, the platform can continuously collect real-time health metrics including heart rate, blood pressure, blood oxygen saturation (SpO2), sleep patterns, step count, and stress levels. This continuous biometric data stream can then be fed into AI models to detect anomalies, generate proactive health alerts, and provide doctors with rich longitudinal patient data for better clinical decision-making.

11.4 Predictive Health Analytics with Custom ML Models

While the current AI features leverage pre-trained large language models, a major future enhancement involves training custom machine learning models on anonymized healthcare data collected through the platform. These models will enable several predictive capabilities:

  • Disease Risk Prediction: Predicting the likelihood of chronic diseases such as diabetes, hypertension, and cardiovascular disease based on patient health records, lifestyle data, and genetic factors.
  • Medication Adherence Prediction: Identifying patients at risk of not following prescribed medication schedules and triggering proactive reminder interventions.
  • Early Warning System: Detecting early warning signs of medical emergencies by analyzing vital sign trends from wearable devices.
  • Appointment No-Show Prediction: Predicting patients who are likely to miss appointments, allowing clinics to manage scheduling more efficiently.

11.5 ABDM / ABHA Health ID Integration

Integration with India's Ayushman Bharat Digital Mission (ABDM) and the Ayushman Bharat Health Account (ABHA) identity system is a high-priority future enhancement. This integration will allow patients to link their Synora Health account with their government-issued ABHA Health ID, enabling interoperability with hospitals, pharmacies, and insurance providers across India's national digital health ecosystem. All health records, prescriptions, and appointment histories created on Synora Health will be accessible through the ABHA system, and records from other ABDM-compliant providers will be automatically imported into the patient's Synora Health profile.

11.6 Multi-Language Support

To make Synora Health accessible to India's diverse population, implementing multi-language support is a critical future enhancement. The planned approach uses i18next — an internationalization framework for React — to provide translations in the following languages as an initial phase: Hindi, Gujarati, Tamil, Bengali, Marathi, Telugu, Kannada, and Malayalam. AI symptom analysis will also be extended to accept input in regional languages using language detection and translation preprocessing before passing to the LLM.

11.7 Online Pharmacy Integration

A seamless pharmacy integration will allow patients to order prescribed medications directly from the prescription module within Synora Health. Partnerships with online pharmacy providers will enable:

  • One-click medicine ordering from AI-generated or doctor-written prescriptions
  • Real-time medicine availability and pricing information
  • Home delivery tracking within the platform
  • Automatic medicine reminder setup upon order confirmation
  • Generic medicine substitution suggestions to reduce patient costs

11.8 Advanced AI Diagnostic Imaging

The current health record AI analysis using Gemini Vision provides general insights from uploaded medical images. A significant enhancement involves training and integrating specialized deep learning models for medical image analysis. Planned capabilities include:

  • X-Ray Analysis: Deep CNN models (ResNet, DenseNet) trained on chest X-ray datasets (CheXNet) to detect pneumonia, tuberculosis, and lung nodules.
  • Skin Lesion Classification: Dermatology AI to classify skin lesions as benign or malignant using dermoscopy images.
  • Diabetic Retinopathy Detection: Automated screening of retinal fundus images for diabetic retinopathy grading.
  • ECG Analysis: AI-powered electrocardiogram analysis for cardiac arrhythmia detection.

11.9 Health Insurance Integration

A dedicated insurance module will enable direct integration with health insurance providers for cashless treatment facilitation. Key features include real-time insurance eligibility verification, automated pre-authorization requests for planned procedures, digital claim submission from within the platform, and claim status tracking. This will eliminate the cumbersome paperwork typically associated with health insurance claims and provide patients with a seamless end-to-end healthcare experience.

11.10 Cloud Deployment and Scalability

For production-scale deployment, the infrastructure will be migrated to a cloud-native architecture using:

  • Docker and Kubernetes: Containerization of all services for consistent deployment and auto-scaling based on traffic load.
  • AWS S3 / Google Cloud Storage: Cloud-based file storage for health records, eliminating local storage limitations.
  • Redis: In-memory caching for frequently accessed data (doctor listings, statistics) to reduce database load.
  • Nginx / AWS ALB: Load balancing across multiple FastAPI instances for high availability.
  • MongoDB Atlas: Managed cloud MongoDB with automated backups, global replication, and HIPAA compliance.
  • Cloudflare: CDN and DDoS protection for the frontend assets.

11.11 Telemedicine for Rural Areas

A specialized module designed for rural healthcare delivery will enable doctors to conduct asynchronous consultations (store-and-forward telemedicine) for patients in areas with poor internet connectivity. Patients can record their symptoms via voice or video, which are then transmitted when connectivity is available. AI-powered automatic transcription and translation will make these consultations accessible in regional languages, significantly extending the reach of quality healthcare to underserved communities.
